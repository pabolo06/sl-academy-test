## 🎯 Guias por Cenário

### Cenário: Primeiro Acesso ao Projeto
1. Leia [README.md](./README.md)
2. Siga [GETTING_STARTED_DEV.md](./GETTING_STARTED_DEV.md)
3. Execute [INICIO_RAPIDO.md](./INICIO_RAPIDO.md)
4. Consulte [COMMANDS.md](./COMMANDS.md)

### Cenário: Fazer Deploy em Produção
1. Leia [DEPLOYMENT_GUIDE.md](./docs/DEPLOYMENT_GUIDE.md)
2. Configure [ENVIRONMENT_VARIABLES.md](./docs/ENVIRONMENT_VARIABLES.md)
3. Execute [DATABASE_MIGRATIONS.md](./docs/DATABASE_MIGRATIONS.md)
4. Configure [MONITORING_AND_ALERTING.md](./docs/MONITORING_AND_ALERTING.md)
5. Configure [BACKUP_AND_RECOVERY.md](./docs/BACKUP_AND_RECOVERY.md)

### Cenário: Responder a um Incidente
1. Siga [OPERATIONS_RUNBOOK.md](./docs/OPERATIONS_RUNBOOK.md) seção "Incident Response"
2. Consulte [MAINTENANCE_GUIDE.md](./MAINTENANCE_GUIDE.md) seção "Resposta a Incidentes"
3. Verifique [MONITORING_AND_ALERTING.md](./docs/MONITORING_AND_ALERTING.md)

### Cenário: Adicionar Nova Funcionalidade
1. Revise [design.md](./.kiro/specs/sl-academy-platform/design.md)
2. Consulte [API_DOCUMENTATION.md](./docs/API_DOCUMENTATION.md)
3. Siga padrões em [GETTING_STARTED_DEV.md](./GETTING_STARTED_DEV.md)
4. Teste com [TESTING_CHECKLIST.md](./TESTING_CHECKLIST.md)

### Cenário: Otimizar Performance
1. Analise [PERFORMANCE_TESTING.md](./docs/PERFORMANCE_TESTING.md)
2. Otimize queries com [QUERY_OPTIMIZATION.md](./docs/QUERY_OPTIMIZATION.md)
3. Implemente cache com [CACHING_STRATEGY.md](./docs/CACHING_STRATEGY.md)
4. Configure CDN com [CDN_CONFIGURATION.md](./docs/CDN_CONFIGURATION.md)

### Cenário: Auditoria de Segurança
1. Execute [SECURITY_AUDIT.md](./docs/SECURITY_AUDIT.md)
2. Rotacione secrets com [SECRETS_MANAGEMENT.md](./docs/SECRETS_MANAGEMENT.md)
3. Revise [SUPABASE_RLS_FIX.md](./SUPABASE_RLS_FIX.md)
4. Atualize [MAINTENANCE_GUIDE.md](./MAINTENANCE_GUIDE.md)

### Cenário: Onboarding de Novo Hospital
1. Siga [USER_GUIDE.md](./docs/USER_GUIDE.md)
2. Configure usando [DEPLOYMENT_GUIDE.md](./docs/DEPLOYMENT_GUIDE.md)
3. Treine usuários com [USER_GUIDE.md](./docs/USER_GUIDE.md)
4. Monitore com [OPERATIONS_RUNBOOK.md](./docs/OPERATIONS_RUNBOOK.md)
